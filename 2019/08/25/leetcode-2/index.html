leetcode-151场周赛总结

发表于 2019-08-25 | 分类于 算法题 ， leetcode

第一次玩周赛，成绩真是不理想，四道题只AC了一道，排名 412/1340，还需努力！

链接：https://leetcode-cn.com/contest/weekly-contest-151

查询无效交易

https://leetcode-cn.com/contest/weekly-contest-151/problems/invalid-transactions/

这道题难度easy，开始题意不理解，绕了很多弯路，后面想清楚了，出现了超时的现象，但是整个代码解法和大神的应该差不多。

附上chensuim的题解：

可以学习的地方在于split

之前split往往会一项项赋值，显得代码非常臃肿，可以直接一行搞定

比较字符串最小字母出现频次

https://leetcode-cn.com/contest/weekly-contest-151/problems/compare-strings-by-frequency-of-the-smallest-character/

唯一一道AC的题目，但是代码显得不简洁，比如

完全可以使用map

collections.Counter、map、bisect 这三个用法可以学习下

从链表中删去总和值为0的连续节点

https://leetcode-cn.com/contest/weekly-contest-151/problems/remove-zero-sum-consecutive-nodes-from-linked-list/

链表的题目已经很久没做了，想法是先把链表转换成list，再做操作

大神题解：

巧妙之处是在于ans=ans[:k]，如果末尾到起点方向k的节点相加为0，则取1:k的节点，抛弃相加为0的节点

然后reversed可以留意学习下

餐盘栈

https://leetcode-cn.com/contest/weekly-contest-151/problems/dinner-plate-stacks/

使用基础数据结构来构造一个新的数据结构，想法有点类似于维护一个list，list里每个都是一个stack

实现起来会有点复杂

大神题解：

heapq这个库可以学习下

希望下次周三名次能提升！
